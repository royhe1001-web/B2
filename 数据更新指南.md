# B2 策略数据更新指南

## 核心原则

**指标计算不是幂等的。** 增量更新时绝对不能截断历史重算指标，必须保持指标序列的连续性。

## 为什么增量更新会破坏数据

`calc_all_indicators` 在一次计算全量数据和分窗口计算时，边界值会不同：
- MA120、MA60 等长周期均线在窗口起始处缺少足够历史，值有偏差
- MACD 的 EMA 递归依赖初始值，截断重算会改变整个序列
- 砖型图的砖块大小依赖历史极值，窗口切换时规则可能变化
- KDJ 的 RSV 滚动计算同样受影响

这些微小偏差传导到 Spring 信号 → 不同的入场判断 → 不同的回测收益。

## 正确的数据更新方式

### 方式一：全量重建（推荐，每季度一次）
```bash
# 1. 下载全量kline数据（tushare）
# 2. 一次性计算所有指标
python build_features.py --full
# 3. 验证回测收益不变
python run_backtest_2026.py
```

### 方式二：仅追加OHLCV，不重算指标（日常，暂不可用）
当前 `calc_all_indicators` 不支持增量模式。
如需日常更新，需改造指标函数支持传入前一状态值。

### 方式三：不回测最新日期（最简单）
如果只是看信号，不需要回测到最新日期：
直接用现有 features 跑回测，结束日期为 features 的最新日期。

## 当前正确做法

1. 从百度网盘下载完整 features 包（全量一体计算）
2. 解压覆盖 `ML_optimization/features/`
3. 直接跑回测，不跑增量更新
4. 每季度重新下载一次完整包更新数据

## 绝对禁止的操作
- 对已有 features 运行 `calc_all_indicators` 增量更新
- 截断历史窗口后重算指标
- 用不同版本 `indicators.py` 混算数据
